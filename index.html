    基于 Docs-as-Code 架构的自动化开发者文档平台 | 个人全栈项目 (2026.02 - 至今)¶

    • 项目角色：技术文档架构 / 独立开发者
    • 技术栈： Markdown, Git, MkDocs, Github Actions (CI/CD), Vale (Linter), Mermaid.js

    • 项目背景：针对传统硬件数据手册（PDF）和软件接口文档维护成本高、协作效率低等痛点，独立从 0 到 1 搭建现代化技术文档发布平台，提升开发者体验（DX）。

    • 核心成果：

      • 文档工程化建设：采用 MkDocs 搭建响应式 Web 站点；配置 GitHub Actions 实现持续集成与自动化部署；引入 Vale 集成 Google/ Microsoft 规范，搭建文档质量自动化门禁，自动检测低级拼写与排版错误。
      • 硬件手册重构 (Hardware)：运用 Mermaid.js 将 SPI 通信协议转化为直观的代码化时序图；使用结构化 Markdown 重构寄存器映射表（Register Map），并提供 C 语言驱动配置示例，降低嵌入式开发者试错成本。
      • RESTful API 指南 (Software)：编写 IoT 设备数据上报接口集成指南，涵盖 OAuth 2.0 鉴权流程，规范化定义 Endpoint、参数速查表及 JSON 响应示例。
