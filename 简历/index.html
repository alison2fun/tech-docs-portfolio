    个人简介

    我是一名电子信息硕士背景的技术文档候选人，关注技术理解、信息结构化、文档工程化和开发者体验。

    过去的学习和项目经历让我比较习惯处理复杂资料：英文论文、硬件规格书、接口说明、产品需求、用户流程和工具链文档。现在我正在把这些能力整理成一个技术文档作品集，用实际页面展示自己如何组织内容、设计阅读路径、写 Quick Start、API Reference、PRD、用户手册和故障排查文档。

    我也长期使用 AI 工具辅助学习、写作和原型开发。对我来说，AI 辅助编程最有价值的地方，不只是生成代码，而是帮助我更快把一个想法做成可以点击、可以试用、可以讨论的产品原型，再基于真实流程反推用户文档、接口说明和产品边界。

    核心能力

    技术理解与资料重构

    • 电子信息硕士背景，具备软硬件技术理解基础；
    • 熟悉硬件规格书、SPI / I2C 通信协议、寄存器说明、底层时序等资料类型；
    • 能将复杂技术资料拆解为结构化文档、表格、流程图和示例代码；
    • 具备英文技术资料阅读能力，曾阅读分析 120+ 篇英文论文。

    文档工程化与 Docs-as-Code

    • 熟练使用 Markdown、Git、MkDocs、GitHub Actions、Mermaid.js；
    • 能搭建静态技术文档站点，并组织导航、侧边栏、页面结构和阅读路径；
    • 能使用 Vale 进行文档质量检查，理解文档规范、术语一致性和自动化检查流程；
    • 具备将文档纳入版本管理和自动部署流程的实践经验。

    API / 开发者文档写作

    • 了解 RESTful API、JSON、OAuth 2.0 鉴权流程；
    • 能编写 Endpoint、请求参数、响应字段、错误码、调用示例等 API 文档内容；
    • 能用 Mermaid 绘制鉴权流程、接口调用链路和系统流程图；
    • 关注开发者接入过程中的信息查找成本和调试成本。

    C 端产品文档与用户手册

    • 能根据产品类型编写用户手册、FAQ、PRD、版本日志和功能说明；
    • 能用面向用户的语言解释产品价值、操作步骤、异常情况和隐私说明；
    • 能结合截图、流程和真实操作路径组织产品帮助文档；
    • 具备小程序产品原型制作和文档整理经验。

    项目背景

    我希望搭建一个面向技术文档岗位的作品集网站，不只是展示简历，也展示自己如何组织技术内容、设计文档结构、维护文档质量和输出不同类型的技术文档。

    因此，我用 MkDocs 搭建了一个技术文档站点，并围绕文档工程化、硬件资料重构、API 文档、小程序产品文档等方向持续补充案例。

    主要工作

    • 配置 GitHub Actions，实现文档站点自动部署；
    • 引入 Vale 文档检查工具，探索写作规范、术语一致性和自动化质量检查；
    • 使用 Mermaid.js 绘制流程图、时序图和接口调用链路；
    • 将作品集从“简历堆叠”重构为更清楚的小型文档系统。

    AI 辅助编程与产品原型制作经验

    我长期使用 ChatGPT、Claude、Gemini、Codex、Tabbit 等 AI 工具辅助学习、写作和开发。

    在做小程序和作品集项目时，我发现 AI 辅助编程最有价值的地方，不只是“帮我写代码”，而是可以把一个想法快速变成可交互的产品原型。

    我做过 3 个 C 端小程序原型：

    • 微步 ACTION：AI 任务拆解小程序，把用户输入的大目标拆成 5 个可执行步骤；
    • 照片换底色神器：照片背景替换工具，支持选择底色、调整尺寸与压缩、保存到相册；
    • PopDots：波点拼贴图片生成器，支持模板、底图、笔刷、盲盒、橡皮擦和导出图片。

    这些项目让我更熟悉一种工作方式：

    先用 AI 和代码工具快速做出可以点击、可以试用、可以截图的原型，再围绕原型讨论产品逻辑、用户流程、异常边界和文档结构。

    这对技术文档工作也很有帮助。很多文档不是在产品完全确定之后才开始写，而是在产品和功能逐步成型的过程中，帮助团队把需求、流程、接口和用户路径说清楚。

    通过这些原型项目，我主要沉淀了几类能力：

    • 把模糊想法拆成可实现的功能模块；
    • 用 AI 辅助完成页面、交互和基础逻辑开发；
    • 通过截图和真实流程反推用户文档结构；
    • 为 C 端产品补充用户手册、FAQ、PRD、API Reference 和版本日志；
    • 把产品原型转化成作品集中的可展示案例。
