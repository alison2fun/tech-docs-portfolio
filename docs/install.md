# 快速开始：搭建本地文档质量检查环境

## 这篇文档适合谁

这篇文档适合希望在本地搭建文档质量检查环境的技术写作者、资料开发工程师或 Docs-as-Code 学习者。

读者需要具备基础命令行操作能力，但不需要提前掌握 Vale 规则语法。完成本文后，你可以在本地运行一次文档质量检查，并理解检查结果的基本含义。

## 完成后你将能够

完成本文后，你将能够：

* 安装并验证 Vale 是否可用；
* 在 VS Code 中安装 Vale 插件；
* 打开一个已配置文档检查规则的项目；
* 运行一次本地文档检查；
* 根据检查结果判断文档是否存在格式、术语或写作风格问题。

## 前置条件

开始前，请确认你已经具备以下环境：

| 条件   | 说明                                         |
| ---- | ------------------------------------------ |
| 操作系统 | Windows / macOS / Linux                    |
| 编辑器  | 已安装 VS Code                                |
| Git  | 已安装 Git，并能克隆或打开项目                          |
| 终端   | 能使用 PowerShell、Terminal 或其他命令行工具           |
| 项目文件 | 当前项目中包含 `.vale.ini`、`styles/` 和 `docs/` 目录 |

## 步骤 1：安装 Vale

### 目标

让本地环境具备文档风格检查能力。

### 操作

根据你的操作系统选择安装方式。

#### Windows

在 PowerShell 中运行：

```powershell
winget install Vale.Vale
```

#### macOS

在 Terminal 中运行：

```bash
brew install vale
```

### 预期结果

安装完成后，运行以下命令检查版本：

```bash
vale -v
```

如果终端输出 Vale 版本号，说明 Vale 已安装成功。

示例输出：

```text
vale version 3.x.x
```

## 步骤 2：安装 VS Code Vale 插件

### 目标

让 VS Code 在编辑 Markdown 文件时显示文档风格检查提示。

### 操作

1. 打开 VS Code；
2. 点击左侧扩展图标；
3. 搜索 `Vale`；
4. 安装 Vale 插件；
5. 重新打开当前文档项目。

### 预期结果

打开 Markdown 文件后，VS Code 可以识别 Vale 配置，并在存在问题的文本处显示提示。

如果暂时没有看到提示，可以先继续完成后面的本地命令行检查。

## 步骤 3：打开文档项目

### 目标

进入已经配置好 Vale 规则的文档项目。

### 操作

在 VS Code 中打开当前作品集项目根目录。

项目根目录下应包含以下文件或目录：

```text
tech-docs-portfolio/
├── docs/
├── styles/
├── .vale.ini
├── mkdocs.yml
└── README.md
```

### 预期结果

你可以在 VS Code 左侧文件树中看到 `.vale.ini`、`styles/` 和 `docs/`。

其中：

* `.vale.ini` 用于指定 Vale 的检查配置；
* `styles/` 用于存放写作规则；
* `docs/` 用于存放 Markdown 文档内容。

## 步骤 4：运行本地文档检查

### 目标

验证本地文档质量检查流程是否可用。

### 操作

在项目根目录运行：

```bash
vale docs/
```

### 预期结果

如果文档中存在格式、术语或写作风格问题，终端会输出对应的文件、行号和规则名称。

示例输出：

```text
docs/index.md
  12:5  error  文档中英文之间缺少空格  MyStyle.Spacing
```

如果没有发现问题，终端可能不会输出错误，或只显示检查完成信息。

## 如何判断检查是否成功

你可以从以下几种情况判断 Vale 是否正常工作：

| 现象               | 含义            |
| ---------------- | ------------- |
| `vale -v` 能输出版本号 | Vale 安装成功     |
| `vale docs/` 能运行 | Vale 可以扫描文档目录 |
| 输出文件名、行号和规则名称    | Vale 成功发现文档问题 |
| 修改问题后再次运行，提示减少   | 检查规则生效        |

## 常见问题

### 运行 `vale -v` 后提示 command not found 怎么办？

可能原因包括：

* Vale 没有安装成功；
* Vale 安装路径没有加入系统 PATH；
* 安装后没有重新打开终端。

建议重新安装 Vale，并重启终端后再次运行：

```bash
vale -v
```

### VS Code 安装插件后没有提示怎么办？

可以检查以下几项：

* 当前打开的是项目根目录，而不是单个 Markdown 文件；
* 项目根目录下存在 `.vale.ini`；
* 当前文件是 Markdown 文件；
* 已经重新加载 VS Code 窗口。

### 本地检查和 GitHub Actions 检查有什么区别？

本地检查用于作者在提交前提前发现问题。
GitHub Actions 检查用于在提交后自动验证文档质量，避免不符合规则的内容进入主分支。

两者配合使用，可以减少人工审查中的重复性工作。
