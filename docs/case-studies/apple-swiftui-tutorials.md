# SwiftUI 教程如何用同一个应用承接四个学习阶段

> **案例性质**：公开文档复盘，不代表我参与过 Apple Developer Documentation 的编写或维护。本文基于 2026 年 7 月可访问页面进行分析。

**分析对象**：[Apple SwiftUI Tutorials](https://developer.apple.com/tutorials/swiftui)

有些教程每章都换一个例子，读者学会了局部语法，却很难看到这些知识怎样进入一个完整应用。SwiftUI 教程选择持续扩展 Landmarks 示例，把新概念留在同一个上下文里。

页面给出了约 4 小时 25 分钟的总时长，并把内容拆成四章。读者在开始前就能判断投入规模，也能按章节安排学习。

## 同一个样例持续生长

第一章从视图组合、列表导航和用户输入开始。后面的绘图与动画、复杂界面、UIKit、watchOS 和 macOS 集成都继续沿用已经熟悉的项目背景。

这种连续性降低了切换成本。读者不必每章重新理解业务场景，可以把注意力放在当前新增的结构和行为上。

## 章节时长也是导航信息

每个小节都标出预计时间，例如 20 分钟、35 分钟或 40 分钟。时间标签看起来很轻，却能帮助读者决定现在开始，还是先保存进度。

章节之外，页面继续提供 API 文档、视频、示例代码和工具下载。教程负责建立路径，参考资料负责查找细节，两者在同一入口下保持联系。

## 连续样例值得保留

连续教程需要一条稳定的样例主线。每章只引入足够支撑当前任务的新概念，也要让读者看见原有成果怎样被扩展。

我还会把预计时间当作内容设计的一部分。它能提醒作者检查章节体量，也能让读者更从容地安排一次学习。
