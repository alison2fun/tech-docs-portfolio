    写作样稿

    我一开始把所有样稿放进一张长表格，信息很全，找起来却不轻松。一个只想看 Quick Start 的人，需要先扫过 PRD、版本日志和产品总览，才能确认自己该点哪里。

    所以这页现在只做一件事：按读者要完成的任务，把已经写完的文档放到对应入口里。模拟项目和个人原型会继续标出来。

    按读者任务查看

    01

    先让读者做成一次

    前置条件、最短路径、验证结果和失败后的恢复，都要在第一次操作里接得上。

    Quick Start搭建本地文档质量检查环境 Troubleshooting文档质量流水线故障排查 Developer Quick Start · 模拟OpenClaw 开发者快速入门
    02

    让文档可以长期维护

    写作规则、自动检查、发布流程和更新记录需要留在同一套工作流里。

    Project Overview文档质量自动化流水线 Style Guide技术文档写作风格指南 WorkflowGitHub Actions 文档检查流程 Changelog文档项目更新记录
    03

    让开发者能够查找和调用

    接口、时序和寄存器信息最终都要回到一次可以执行、可以验证的技术任务。

    API Guide · 模拟IoT 接口集成指南 Hardware Reference · 模拟XYZ-2024 数据手册重构 API Reference · 原型微步 ACTION API 参考
    04

    把产品流程交给用户和协作者

    同一个产品需要面向不同读者解释范围、操作、接口、变化和已知限制。

    Product Overview三个小程序产品总览 User Guide微步 ACTION 用户文档 PRD · 原型微步 ACTION 产品需求文档 Release Notes · 原型微步 ACTION 版本日志 Screenshot GuidePopDots 波点拼贴帮助 Screenshot Guide照片换底色小程序帮助

    如果你只看两篇

    想看任务型文档，可以先打开搭建本地文档质量检查环境。它从安装条件开始，一直写到第一次检查成功和常见报错。

    想看开发者文档，可以继续看IoT 接口集成指南。这是一份明确标注为模拟场景的样稿，重点放在认证、请求、响应、错误处理和调试路径怎样连成一次完整调用。

    关于英文样稿

    英文 Quick Start 和英文项目概览仍在计划中。完成之前，我不会在这里放一个点进去没有内容的入口。现有英文资料阅读经历可以在关于我和简历中查看。
